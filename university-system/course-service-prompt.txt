Generate a complete Spring Boot project (module :course-service) for a backend service called course-service under university-system.

Goal:
Create a Course Service application that provides CRUD operations for courses, connects to PostgreSQL database named courseDB, and includes a Dockerfile.

This is for teaching backend development, so keep the code clean, simple, and beginner-friendly.

The service must also communicate with student-service in two ways:
1. When listing students enrolled in a course, use RestTemplate to call student-service over HTTP and retrieve the student details.
2. When enrolling a student into a course, first validate that the course exists in courseDB, then validate that the student exists by calling student-service using Feign Client (GET by id).

Make sure the root packaging is: az.edu.ada.wm2

2. Enrollment support
The course-service must support enrollments between courses and students.

You may implement enrollment in a simple teaching-friendly way using a separate Enrollment entity or a course_student table equivalent.

Suggested Enrollment fields:
- Long id
- Long courseId
- Long studentId

Store only IDs in course-service database.
Do NOT duplicate student data in course-service DB.

Enrollment requirements:
- A student can be enrolled into a course
- A course can have many students
- One student can be enrolled in many courses
- Prevent duplicate enrollment of the same student into the same course

Provide endpoints such as:
- POST /api/v1/courses/{courseId}/students/{studentId}
  -> enroll a student into a course
- GET /api/v1/courses/{courseId}/students
  -> list detailed student info for students enrolled in that course

3. HTTP integration with student-service

A. RestTemplate usage
When calling:
- GET /api/v1/courses/{courseId}/students

The course-service should:
- validate that the course exists in courseDB
- retrieve all enrolled studentIds for that course from courseDB
- for each studentId, call student-service using RestTemplate:
  - GET /api/v1/students/{id}
- collect the returned student details and return them as response

Create:
- a RestTemplate configuration class under config package
- a DTO representing student data received from student-service

B. Feign Client usage
When calling:
- POST /api/v1/courses/{courseId}/students/{studentId}

The course-service should:
- first check if the course exists in courseDB
- then call student-service using Feign Client:
  - GET /api/v1/students/{id}
- if student exists, save the enrollment
- if student does not exist, return a proper error response
- if enrollment already exists, return a meaningful error response

Create:
- a Feign client interface under client package
- annotate the application with @EnableFeignClients

Student-service base URL:
- make it configurable in application.properties
- use a property such as:
  student.service.base-url=http://localhost:8080

Assume student-service endpoint:
- GET /api/v1/students/{id}
